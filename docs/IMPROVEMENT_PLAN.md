# [IMPROVEMENT PLAN] 시스템 고도화 로드맵

시스템의 견고함(Resilience)과 유지보수성(Maintainability)을 시니어 아키텍트 수준으로 끌어올리기 위한 단계별 개선 계획입니다.

---

## 1단계: 설정 관리 및 모델 고도화 (Config & Model Refactoring)

**목적:** 하드코딩된 설정을 타입 안전(Type-safe)하게 관리하고 팩토리 패턴을 통한 객체 생성 최적화.

* **`SourceConfig` 모델 도입**: `config.py`에 Pydantic 모델을 정의하여 `SOURCES` 데이터의 유효성을 강제하고 자동 완성을 지원합니다.
* **Scraper Factory 개선**: `app.py`의 `_create_scraper` 로직을 설정 기반의 동적 매핑으로 변경하여 새로운 소스 추가 시 코드 수정을 최소화합니다.

## 2단계: 스크래퍼 코드 중복 제거 및 추상화 (LawGoKrScraper Abstraction) ✅

**목적:** 유사한 사이트(law.go.kr) 간의 중복 로직을 통합하여 유지보수 효율 증대.

* **`LawGoKrScraper` 기차(Base) 클래스 생성**: `base.py`와 실제 스크래퍼 사이에 법제처 전용 공통 레이어를 도입합니다.
* **JS 추출 로직 일원화**: `StatuteScraper`와 `AdminRuleScraper`에서 중복되는 DOM 순회(JS) 및 계층 구조 파싱 로직을 공통화합니다.

## 3단계: 데이터 품질 가드 및 테스트 자동화 (Testing & Quality Guard) ✅

**목적:** RAG의 기반이 되는 법령 데이터의 무결성을 보장하고 퇴행(Regression) 방지.

* **유틸리티 유닛 테스트 작성**: `utils/text.py`의 클리닝 로직에 대해 다양한 법령 케이스(항/호/목 파편화, 인덴트 보존)를 포함한 7개의 테스트 구축 완료.
* **검증 스크립트 고도화**: `tests/test_db_quality.py`를 통해 DB 스키마, 데이터 유무, 컨텐츠 품질(길이, 필수 필드), 중복 여부를 자동 검증하도록 통합 완료.

## 4단계: 운영 복원력 및 체크포인트 (Resilience & Checkpoint) ✅

**목적:** 대규모 데이터 수집 시 장애(네트워크 끊김, 브라우저 다운)에 대응하는 복구 능력 확보.

* **체크포인트 기록**: `scrape_runs` 테이블을 활용하여 현재 수집 중인 위치를 실시간으로 저장합니다.
* **Resume(이어서 수집) 기능**: 수집 중단 시 마지막으로 성공한 조문 이후부터 수집을 재개하는 로직을 구현합니다.

---

## 📅 실행 로드맵

위 단계는 상호 의존성을 고려하여 **1 -> 2 -> 3 -> 4** 순서로 차례대로 진행될 예정입니다.
현재 각 단계의 설계안을 수립 중이며, 승인 시 1단계부터 즉시 착수하겠습니다.
